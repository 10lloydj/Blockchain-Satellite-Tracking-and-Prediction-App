We wish to explore the potential applications blockchain technology can have within the space industry.
The ever-growing constellations of satellites such as Starlink and Amazons' Project Kuiper pose a huge concern to other space missions and orbiting artificial satellites.

My app addresses these issues by providng a decentralised platform to freely inform the greater community of satellite data; allowing operators to manoeuvre confidently around the Earths orbit.

Ethereum Virtual Machine (EVM)

Allows smart contract development & stores the blockchain

The EVM is the global virtual computer where all user's on the Ethereum network stores their data in blocks.
Smart contracts are published onto the EVM network to be compiled and cause changes to the EVM state, forming a blockchain.

My application uses a smart contract to record satellite data at that current time; creating a 'block' onto the EVM. In addition, allows the user's to view historic recordings of satellite data that are saved in blocks. Thus, forming a connection of blocks of all the satellite data on the network (blockchain).

This allows operators to record a selection of satellites through time and look back at their positions and other relevant data.

N2yo

Satellite data API

The API provides data for developers to build satellite tracking or prediction applications. It is a free but transaction limited REST API.

The API identifies satellites by their NORAD ID number; an integer ID identifying artificial satellites generated by United States Space Command (USSPACECOM).

My application retrieves the position relevant data and the TLE (Two Line Element) which identifies satellites manoeuvring statistics. This is essentially used to track satellites in real-time and as input to the Skyfield API.

Skyfield

An API using NASA's mathematical model to predict satellite data

Astronomy API that computes positions of stars, planets and artificial satellites. This API is much more advanced than N2yo, given the mathematical calculations used within various functions.

My application uses a mathematical model built by NASA scientists that uses the TLE to predict a satellite's location. The model SGP4 is a very complex model that is continuously being innovated to improve accuracy. Skyfield has imported this model to python; allowing developers to accurately predict positions at certain times in the future.
